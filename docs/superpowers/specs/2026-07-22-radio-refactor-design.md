# Internet Radio PWA — Рефакторинг

**Дата**: 2026-07-22  
**Статус**: Утверждён

## Цель

Рефакторинг PWA интернет-радио плеера: загрузка станций с API Radio Record, новый мобильный UI в тёмной теме, перетаскивание станций, авто-переключение качества потока, получение названия текущего трека.

## Архитектура

Проект разбивается на модули (Подход B):

```
index.html          — минимальная разметка (контейнеры)
css/style.css       — все стили, тёмная тема, mobile-first
js/
  api.js            — запросы к API Radio Record
  player.js         — управление Audio, авто-переключение потоков
  ui.js             — рендер, drag & drop, модальное окно, события
  app.js            — инициализация, polling now-playing, координация
sw.js               — Service Worker (кеширование)
manifest.json       — PWA манифест (без эмодзи в иконках)
```

## Данные и API

### Станции
- **Источник**: `GET https://www.radiorecord.ru/api/stations`
- **Структура**: `result.stations[]` — поля: `id`, `title`, `tooltip`, `svg_fill`, `svg_outline`, `stream_64`, `stream_128`, `stream_320`, `stream_hls`, `bg_color`, `genre`
- **Загрузка**: при старте приложения однократно
- **Хранение**: список станций в памяти, порядок и свои станции в `localStorage`

### Текущий трек
- **Источник**: `GET https://www.radiorecord.ru/api/stations/now/`
- **Структура**: массив `[{id: stationId, track: {artist, song, image100/200/600}}]`
- **Матчинг**: по `id` станции с текущей играющей
- **Polling**: каждые 10 секунд
- **Отображение**: `ARTIST — SONG` в track bar

### Потоки и авто-переключение
- Приоритет: `stream_320` → `stream_128` → `stream_64`
- При буферизации/ошибке — переход на следующий уровень
- Использовать событие `waiting` / `stalled` на Audio элементе

### Свои станции
- Кнопка `+` в header → модальное окно с полем URL
- Из URL извлекается домен как название
- Плашка без SVG (буквенный аватар — первая буква названия)
- Сохраняются в `localStorage` отдельно от станций API

### Порядок станций
- Свои станции всегда в начале списка
- Drag & drop переупорядочивает станции API
- Порядок сохраняется в `localStorage`

## Компоненты

### Header
- Текст «Интернет Радио»
- Кнопка `+` (добавить свою станцию)

### Track Bar
- Artist — Song (жирный, усечение с многоточием)
- Название станции (мелкий, серый)

### Сетка станций (3 колонки)
- Карточка: SVG иконка (36×36) + название (10px, до 2 строк)
- Активная: акцентная обводка + фон `#1e1e30`
- Drag & drop: touch/pointer события, без визуальных ручек
- Свои станции: буквенный аватар вместо SVG

### Bottom Bar (fixed)
- Слайдер громкости с иконками динамика
- Кнопки: previous / play-pause / next
- Play — круглая акцентная кнопка

### Модальное окно
- Затемнённый фон
- Поле ввода URL + кнопка «Добавить»
- Закрытие по клику вне окна

## Модули

### api.js
```js
fetchStations()           // → Promise<Station[]>
fetchNowPlaying()         // → Promise<NowPlaying[]>
```

### player.js
```js
play(station)             // начать воспроизведение
pause()                   // пауза
resume()                  // продолжить
stop()                    // остановить
next() / prev()           // переключение станции
setVolume(v)              // громкость 0..1
onTrackUpdate(callback)   // событие смены трека
// Внутреннее: авто-переключение stream_320→128→64
```

### ui.js
```js
renderGrid(stations, activeId)  // отрисовать сетку
updateTrackBar(artist, song, stationName)
setPlayingState(isPlaying)
showAddModal() / hideAddModal()
initDragDrop(container, onReorder)
```

### app.js
```js
init()                    // загрузка API, рендер, запуск polling
```

## Что удаляется из текущего кода
- Эмодзи (📻) из заголовка, манифеста, иконок
- Старая станция по умолчанию Piano Solo Radio
- Кнопка «Установить приложение» (PWA установка остаётся через beforeinstallprompt, но без большой зелёной кнопки)
- Инлайн-стили и скрипты из index.html
- Поле ввода URL станции на главном экране
- Старые кнопки управления и списка станций

## PWA
- Service Worker: кеширование index.html, css, js
- Manifest: без эмодзи, SVG иконка геометрическая
- beforeinstallprompt: сохраняется, но без явной кнопки (браузер сам предложит)

## Не входит в scope
- Тёмная/светлая тема (только тёмная)
- Поиск/фильтрация станций
- Избранное
- Фоновое воспроизведение через Media Session (оставить из текущего кода где применимо)
